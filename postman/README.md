# Postman Collection – Qeran API

This folder contains a **Postman collection** with all project API endpoints, generated by [Scribe](https://scribe.knuckles.wtf/).

---

## How to import into Postman

### Option A: Import from file (recommended)

1. Open **Postman** (desktop or web).
2. Click **Import** (top left).
3. Choose **Upload Files** or drag and drop:
   - **File:** `postman/qeran-api.postman_collection.json`
4. Click **Import**.
5. The collection **Laravel API Documentation** will appear in your sidebar.

### Option B: Import from URL (after docs are deployed)

1. Log in to your API docs (if protected):  
   `https://your-domain.com/docs/login`
2. Open this URL in the browser (while logged in):  
   `https://your-domain.com/docs.postman`
3. The browser will download `collection.json`.
4. In Postman: **Import** → **Upload Files** → select the downloaded file.

### Option C: Regenerate and use storage file

1. In the project root run:
   ```bash
   php artisan scribe:generate
   ```
2. Import in Postman from:  
   `storage/app/scribe/collection.json`  
   (Import → Upload Files → select that file.)

---

## After importing

1. **Set base URL**  
   The collection uses a variable `baseUrl` (default: `http://127.0.0.1:8000`).  
   - Click the collection → **Variables** tab.  
   - Set **Current value** to your API base (e.g. `https://qeran.app` or `http://127.0.0.1:8000`).

2. **Authentication**  
   Endpoints that need a Bearer token:  
   - Collection → **Authorization** → Type: **Bearer Token** → paste your token,  
   - or set an **Authorization** header: `Bearer {your_token}`.

3. **Re-generate after API changes**  
   When you add or change API routes, run:
   ```bash
   php artisan scribe:generate
   ```
   Then re-import the collection from `postman/qeran-api.postman_collection.json` (or copy `storage/app/scribe/collection.json` over that file and re-import).

---

## Collection contents

The collection includes all documented API routes (e.g. auth, invitations, notifications, profile, settings). Grouping and names follow the Scribe output from your `api/*` routes.
